# Phase 1 Plan

Phase 1 的目标是补齐现代 C++ / Python 工程化能力，重点不是刷题，而是建立稳定的项目开发、测试、提交和复盘习惯。

## Day 1：初始化学习总仓库

目标：

- 创建 `learning-monorepo`
- 建立 GitHub 远程仓库
- 添加 README
- 添加学习日志模板
- 添加 Issue 模板
- 完成第一次 commit 和 push

验收标准：

- GitHub 上能看到仓库和基础文件
- 本地 `git status` 干净
- 至少有一次有效 commit

## Day 2：本地开发工具链盘点与验证

目标：

- 检查 Git、Python、C++ 编译器、CMake、编辑器
- 记录当前环境版本
- 建立工具链检查文档

验收标准：

- 能输出所有工具版本
- 能记录缺失工具
- 能形成 `docs/tooling/environment_check.md`

## Day 3：Git 基础工作流训练

目标：

- 练习 branch、commit、diff、log、restore
- 理解工作区、暂存区、本地仓库、远程仓库

验收标准：

- 能创建并合并一个练习分支
- 能解释 `git status`、`git diff`、`git log` 的作用

## Day 4：README 与文档工程化

目标：

- 让 README 具备项目入口能力
- 建立统一文档风格

验收标准：

- README 能让陌生人理解仓库用途
- 文档目录结构清晰

## Day 5：第一次周内小复盘

目标：

- 复盘 Day 1–Day 4 的提交、问题和知识漏洞
- 调整下一周节奏

验收标准：

- 形成一篇周复盘文档
- 明确下周 5 天任务方向
