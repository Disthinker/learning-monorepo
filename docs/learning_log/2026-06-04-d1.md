## 3. 遇到的问题

### 问题 1：Git 不追踪空目录

现象：`projects/` 下的空目录创建后，GitHub 远程仓库中看不到。

原因：Git 默认不追踪空目录，只追踪文件。

解决方式：在每个空目录中添加 `.gitkeep` 文件。

验证结果：`projects/cpp/.gitkeep`、`projects/python/.gitkeep`、`projects/game_dev/.gitkeep`、`projects/ml_fl/.gitkeep`、`projects/ai_agent/.gitkeep` 已提交。

## 4. 今日学到的关键点

1. Git 仓库不仅保存代码，也可以保存学习路线、日志、模板和工程记录。
2. README 是项目入口，应该让陌生人快速理解仓库用途和结构。
3. Commit message 应该描述本次变更的类型和目的，例如 `docs:`、`chore:`。
4. Git 不会追踪空目录，需要用 `.gitkeep` 保留目录结构。
5. Markdown 模板需要保持可复用，不能残留某一天的固定内容。
6. 工具链盘点的目标不是一次性安装所有工具，而是准确记录当前环境状态。
