## 3. 今日遇到的问题

### 问题 1：对未跟踪文件使用 `git restore` 失败

现象：对新建的 `docs/git/git_workflow_notes.md` 执行 `git restore` 时出现 pathspec 错误。

原因：该文件当时还没有被 Git 跟踪，`git restore <file>` 只能恢复 Git 已知的 tracked 文件。

解决方式：改用 README 这种已被 Git 跟踪的文件练习 `git restore`。

验证结果：理解了 untracked 文件和 tracked 文件的区别。

### 问题 2：临时练习行被提交到远程

现象：`TEMP LINE FOR RESTORE PRACTICE` 出现在远程仓库的 Git 笔记文件末尾。

原因：提交前没有再次检查 `git diff` 和文件内容。

解决方式：删除临时行，并补充修正 commit。

验证结果：本次修正后重新检查。

## 4. 今日学到的关键点

1. `git diff` 默认不显示 untracked 新文件的内容。
2. `git restore <file>` 主要用于恢复已被 Git 跟踪的文件。
3. 新建文件需要先 `git add`，Git 才会把它纳入版本控制。
4. 提交前必须检查 `git status` 和 `git diff`。
5. 分支练习的完整闭环是：创建分支、修改、提交、切回 main、merge、push。
